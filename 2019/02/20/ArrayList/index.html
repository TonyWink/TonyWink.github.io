ArrayList源码分析

发表于 2019-02-20 | |

ArrayList是在工作中最常用的数据结构之一，和linkedList经常一起出现，我的上一篇博文分析了一下LinkedList的相关源码，这边也分析一下ArrayList的源码，并且进行比较。

ArrayList实现的相关接口

ArrayList的实现类主要实现了4个接口

List：提供list增删改查等基本操作
RandomAccess：这是一个标记接口，表示它能够快速随机访问存储的元素
Cloneable：表示支持Object的clone()方法
Serializable：表示支持对象的序列化操作

LinkedList未实现RandomAccess，因此获取元素时使用的是迭代器方式，通过地址来查找元素

那为什么LinkedList不能也用get方式呢，因为它是基于双向链表实现的，查询效率很低，其实现原理在上一篇博文LinkedList源码解析中提到，为了优化查询速度，LinkedList根据索引确定循环从前开始查找还是循环从后开始查找到具体的索引位置，查询效率很低。

ArrayList的新增:

向数组内添加元素时，先判断和确认数组容量是否足够，容量不足则先完成扩容操作，在扩容操作时，只有以空构造函数初始化的，即值为DEFAULTCAPACITY_EMPTY_ELEMENTDATA 扩容时才会给定最小的初始化大小10，若大于10则以指定的大小为准。

ArrayList的扩容策略

ArrayList默认将容量扩大1.5倍，容量不够使用则扩容到所需的大小，扩容后的大小超过最大限制数组大小，设置为最大整数。
